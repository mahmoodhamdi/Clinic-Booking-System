## Issues Found During Testing

### ISSUE-001: Wrong Application on Port (Environment Issue - Not a Bug)
**Type:** Testing Environment Issue
**Description:** During initial testing, a different Next.js project ("Localhost-Tunnel") was running on port 3001, causing confusion with branding.
**Resolution:** Killed the wrong process and started the correct Clinic Booking System frontend.
**Status:** Resolved during testing

### ISSUE-002: Locale Prefix Routes Return 404 (Expected Behavior)
**Type:** Documentation/Clarification
**Description:** The application uses cookie-based locale detection, NOT URL-based locale prefixes.
**Design:** Routes like `/en/dashboard` or `/ar/dashboard` are not supported by design.
**How Language Switching Works:**
1. User clicks language switcher dropdown
2. Cookie `locale` is set to selected language
3. Page reloads with new locale from cookie
**Status:** Working as designed

### ISSUE-003: Language Switcher Works Correctly
**Type:** False Positive
**Description:** The language switcher dropdown functions correctly. Initial testing issues were due to Puppeteer automation timing.
**Evidence:** Screenshot 18 shows dropdown with "العربية" and "English" options visible.
**Status:** No bug - feature works correctly

### ISSUE-004: CORS Configuration (Environment Setup)
**Type:** Environment Configuration
**Description:** FRONTEND_URL in .env needed to match the actual frontend port (3001).
**Fix Applied:** Updated FRONTEND_URL=http://localhost:3001 in .env
**Note:** This is environment-specific configuration, not a code bug.
