# Support Plans — خطط الدعم

> ثلاث خطط دعم مدفوع شهري أو سنوي، كل واحدة موجهة لحجم وأهمية مختلفة من
> العيادات. أي خطة فيها bug-fix warranty مجاني للـ scope الأصلي خلال فترة
> محددة، وبعد كده الدعم بيكون مدفوع بحسب الـ tier.

---

## Standard — للعيادات الصغيرة الجديدة

**$50 / شهر** (أو $500 / سنة، توفير شهرين)

### Scope
- **Email support**: رد خلال **48 ساعة عمل** (الأحد–الخميس، 9 ص – 6 م)
- **Bug fixes** لأي bug جوهري في الـ scope الأصلي (مجاني)
- **Security patches**: أي CVE high/critical في الـ dependencies بيتعالج
  خلال 7 أيام (مجاني)
- **Up to 2 hours** من شغل تطوير شهرياً (للـ small enhancements، تعديل texts،
  تغيير ألوان، إلخ.) — ساعات إضافية بـ $35/ساعة
- **Backup verification quarterly**: مرة كل 3 شهور بنعمل restore drill،
  بنرسل screenshot وreport قصير

### مش في الـ scope
- ❌ Features جديدة كاملة (بـ scope منفصل)
- ❌ Migration من نظام آخر (لو محتاج)
- ❌ On-site visit (Egypt office)
- ❌ Custom integrations
- ❌ Weekend / outside-hours response

### SLA
| الحدث | الـ response المضمون |
|------|---------------------|
| النظام نازل تماماً (down) | 4 ساعات عمل |
| Bug عام بيعطل feature أساسي | 24 ساعة عمل |
| Bug تجميلي / minor | 48 ساعة عمل |
| سؤال أو request للـ enhancement | 72 ساعة عمل |

## Priority — الأكثر طلباً

**$150 / شهر** (أو $1,500 / سنة، توفير شهرين)

كل اللي في Standard، زائد:

### Scope مكتوبة بالكامل
- **Email + WhatsApp** support: رد خلال **24 ساعة عمل** للأسئلة العامة،
  **4 ساعات** للـ blockers
- **Weekend response** ممكن للـ blockers الحرجة (نازل الـ system)
- **5 hours** شغل تطوير شهري شامل في السعر — بيشمل enhancements، content
  updates، report tweaks. ساعات زيادة بـ $30/ساعة (خصم vs Standard)
- **Monthly health check**: مرة في الشهر، بنبعت report:
  - test suite running clean ✅
  - vulnerabilities scan ✅
  - backup restore drill ✅
  - performance baseline ✅
- **Custom integrations small-scope**: webhook setup، Google Calendar sync،
  WhatsApp notifications template — حالات شائعة بتدخل في الـ 5 hours
- **Quarterly review call** (Zoom 30 دقيقة): مراجعة الـ usage stats، اقتراحات
  features، rebalance للـ resources لو الـ usage تغير

### SLA
| الحدث | الـ response المضمون |
|------|---------------------|
| النظام نازل تماماً | **2 ساعة** (24/7) |
| Bug عام بيعطل feature أساسي | 8 ساعات عمل |
| Bug تجميلي / minor | 24 ساعة عمل |
| سؤال أو enhancement request | 48 ساعة عمل |

### Uptime guarantee
- **99% monthly uptime** (max ~7 ساعات downtime مسموحة في الشهر)
- لو نزلنا تحت ده، شهر دعم مجاني تلقائياً (credit)

## Enterprise — للسلاسل والعيادات الـ multi-branch

**$400 / شهر** (أو $4,000 / سنة، توفير شهرين)
**Custom: $600+ / شهر** لو عدد الفروع > 5 أو فيه integrations كبيرة

كل اللي في Priority، زائد:

### Scope كاملة
- **Dedicated WhatsApp channel** بـ response **خلال ساعة** للـ urgent
  (وقت العمل) و **3 ساعات** خارج الـ working hours
- **Phone support** للـ blockers الحرجة (24/7 hotline)
- **15 hours** شغل تطوير شهري شامل، rollover للشهر اللي بعده (مش بيتفقد)
- **On-site visit** كل 3 شهور (لو العميل في القاهرة الكبرى — سفر برة بـ
  travel cost separate)
- **Custom integrations**: ربط مع insurance providers، lab APIs، نظام
  محاسبة موجود (QuickBooks / Xero / Zoho) — scope بيتفاوض
- **Multi-branch support**: تعديلات الـ schema والـ UI للـ multi-clinic،
  branch reports separate
- **White-label**: اسم النظام بتاع العميل، domain خاص، الـ branding كامل لهم
- **24/7 on-call** للـ outages — paging via WhatsApp / phone
- **Quarterly business review** (Zoom 60 دقيقة): metrics، roadmap input،
  capacity planning للسنة جاية

### SLA
| الحدث | الـ response المضمون |
|------|---------------------|
| النظام نازل تماماً | **1 ساعة (24/7)** |
| Bug عام بيعطل feature أساسي | 4 ساعات (24/7) |
| Bug تجميلي / minor | 12 ساعة عمل |
| سؤال أو enhancement request | 24 ساعة عمل |

### Uptime guarantee
- **99.5% monthly uptime** (max ~3.5 ساعات downtime مسموحة شهرياً)
- **99.9% on Enterprise+** (custom contract) — يبقى max ~45 دقيقة شهرياً
- نزلنا تحت ده ➡ pro-rated credit + إجراء RCA (root cause analysis) مكتوب

### الـ deliverables الإضافية للـ Enterprise
- **Compliance documentation pack**: DPA، information security policy summary،
  data flow diagram، backup/recovery plan — لازم لو العميل بيتدقق من
  جهة منظمة (وزارة الصحة، تأمين)
- **Custom training** للموظفين الجدد (بدون cap على عدد الجلسات سنوياً)
- **Hardware recommendations** للسكرتارية (printer للـ roshetta، monitor قياس مناسب)

## ملاحظات عامة على كل الخطط

### مدة الـ warranty المجاني (مرة واحدة، مش متجدد)
- **Basic tier installation** ➡ 3 شهور warranty مجاني لأي bug جوهري في الـ
  scope الأصلي
- **Pro tier installation** ➡ 6 شهور warranty
- **Enterprise tier installation** ➡ 12 شهر warranty

بعد فترة الـ warranty، أي bug خارج الـ support hours المتعاقد عليها بيتحاسب
بالساعة بالـ rate بتاع الـ tier.

### إيش يعتبر "bug جوهري" بدخل في الـ warranty
- crash، 500 errors، broken feature في scope الـ delivery
- regression بسبب dependency security patch
- bug في الـ authorization (RBAC) بيخلي مستخدم يشوف بيانات مش بتاعته
- bug في الـ payment أو scheduling بيخلي double-booking أو wrong total

### إيش مش "bug" بدخل في الـ warranty
- "feature لو موجودة كانت أفضل" — ده enhancement مش bug
- بطء بسبب hosting أقل من recommended — recommendation: ترقية الـ VPS
- مشاكل بسبب custom code العميل ضافه بنفسه

### إلغاء الاشتراك
- شهري ➡ ممكن تلغي في أي وقت، بـ notice 14 يوم
- سنوي ➡ pro-rated refund للشهور المتبقية (بعد الـ first 60 يوم)
- في حالات الـ chargeback / non-payment ➡ السيرفر مش بيقفل، لكن الـ support
  بيتوقف ولحد 90 يوم بـ pay-back لاسترجاع الـ access

### Escalation
| المستوى | متى | مين |
|---------|-----|------|
| L1 | كل الـ tickets العادية | Support engineer on duty |
| L2 | لو L1 ما حلش في الـ SLA window | Senior engineer (نفس اليوم) |
| L3 | outage أو data loss | Founder direct (Mahmoud Hamdy) |

كل العملاء عندهم رقم تليفون L3 — استخدامه بس للـ emergencies.
